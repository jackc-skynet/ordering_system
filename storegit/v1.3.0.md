# Change Set Summary: v1.3.0 - Multi-Order Tracking Support

**Date**: 2026-03-19
**Version**: v1.3.0

## Affected Files
- `src/App.jsx`: Refactored state management and UI components to support multiple active orders.

## Key Changes
- **Multi-Order Core**: Changed `orderResult` to an `activeOrders` array. Customers can now place subsequent orders without losing track of previous ones.
- **Order Switcher UI**: Added a row of order ID buttons in the `OrderTrackingView` to allow users to toggle between different meal progress screens.
- **Aggregate Status FAB**: The Floating Action Button now summarizes all ongoing orders (e.g., "2 份訂單 請取餐！") and pulses if any single order reaches the `ready` state.
- **Persistent State**: Orders remain in the tracking list until explicitly cleared by the user, ensuring full transparency across multiple transactions.

## Commit Message
`v1.3.0 - Add support for tracking multiple concurrent orders with switcher UI`
