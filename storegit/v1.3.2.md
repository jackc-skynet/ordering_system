# Change Set Summary: v1.3.2 - Order Switcher State Synchronization Fix

**Date**: 2026-03-19
**Version**: v1.3.2

## Affected Files
- `src/App.jsx`: Added `key={currentOrder.id}` to the `OrderTrackingView` component call.

## Key Changes
- **State Synchronization Fix**: Resolved a critical bug where `OrderTrackingView` failed to reset its internal `status` state when switching between different active orders. 
- **Component Lifecycle Optimization**: By using a unique `key` based on the order ID, React now correctly unmounts and remounts the tracking view when the selection changes, ensuring all real-time subscriptions and visual indicators start fresh for the selected order.

## Commit Message
`v1.3.2 - Fix: Ensure OrderTrackingView state resets when switching between orders`
