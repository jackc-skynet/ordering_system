# Change Set Summary: v1.3.3 - Centralized Real-time Status Tracking

**Date**: 2026-03-19
**Version**: v1.3.3

## Affected Files
- `src/App.jsx`: Centralized all order status listening to the main `App` component. Simplified child components to rely on the shared `activeOrders` state.

## Key Changes
- **Centralized Synchronization**: Moved all Supabase Realtime status listeners to the parent `App` component. This ensures that any change in an order's status is immediately updated in the global `activeOrders` array.
- **Improved Data Consistency**: By managing status at the top level, we eliminate state divergence between the `OrderTrackingView` and the `FloatingStatusButton`. 
- **Reliable Swapping**: Switching between multiple active orders now correctly displays the most recent status without waiting for new events, as the data source is centrally maintained.
- **Redundancy Reduction**: Removed redundant local status states and individual subscriptions from child components, streamlining the application's real-time architecture.

## Commit Message
`v1.3.3 - Fix: Centralize real-time status tracking to ensure consistency across multiple orders`
